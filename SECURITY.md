# Security Policy

## Supported Versions

Currently supported versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 1.1.x   | :white_check_mark: |
| 1.0.x   | :x:                |
| < 1.0   | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within TsMusic, please follow these steps:

### 1. Do NOT Open a Public Issue

Security vulnerabilities should not be reported through public GitHub issues.

### 2. Email the Maintainers

Send an email to **security@tsmusic.app** with:

- **Subject:** `Security Vulnerability: [Brief Description]`
- **Description of the vulnerability**
- **Steps to reproduce** (if applicable)
- **Possible impact**
- **Suggested fix** (if any)
- **Your contact information** for follow-up

### 3. Response Timeline

- **Acknowledgment:** Within 48 hours
- **Initial assessment:** Within 5 days
- **Fix and release:** Timeline depends on severity
  - Critical: 7 days
  - High: 14 days
  - Medium: 30 days
  - Low: Next scheduled release

### 4. Disclosure Policy

Once the vulnerability is fixed:

- You will receive credit in the security advisory (unless you prefer to remain anonymous)
- A security advisory will be published on GitHub
- The fix will be mentioned in release notes

## Security Best Practices for Users

1. **Keep the app updated** to the latest version
2. **Only download music** from legal sources
3. **Be cautious** when granting storage permissions
4. **Report suspicious behavior** immediately

## Security Measures in TsMusic

- All audio files are scanned before playback
- Network requests use HTTPS
- Local database is encrypted
- No analytics or tracking without user consent

Thank you for helping keep TsMusic secure! 🔒
